# v1.1 功能增强设计文档

> 日期：2026-06-07 | 状态：已批准

## 概述

在 v1.0 基础上新增三个小功能，不涉及数据库表结构变更。

## 改动清单

### 1. 微信/支付宝缴费方式

**前端改动**：`Billing.vue` — 缴费方式下拉框增加"微信""支付宝"选项

**后端改动**：无。`PaymentRequest.paymentMethod` 是自由字符串，`payments.payment_method` 无枚举约束。

### 2. 各村收缴率排行榜

**新增 API**：`GET /api/reports/village-collection-rate`

**参数**：`year` (必填), `month` (必填)

**逻辑**：查 `water_bills` → left join `households` 获取村名 → 按村分组 → 计算应收总额/实收总额/收缴率 → 按收缴率升序（差的排前面）→ 返回列表

**前端**：`Dashboard.vue` — 表格卡片展示

### 3. 异常抄表提醒列表

**新增 API**：`GET /api/readings/abnormal`

**参数**：`limit` (选填，默认 20)

**逻辑**：查 `readings` where `is_abnormal = true` → 关联 `households` 获取户名/村名 → 按 `reading_date` 降序 → limit

**前端**：`Dashboard.vue` — 列表卡片展示，每行显示日期、户名、村名、异常原因

## 仪表盘新布局

```
第一行：四个统计卡片（总户数/本月应收/本月实收/收缴率）— 保持不变
第二行：
  左侧 60%：各村收缴率排行表格（村名/应收/实收/收缴率/进度条）
  右侧 40%：异常抄表提醒列表（日期/户名/村名/异常原因）
```

## 异常抄表查询 SQL

```sql
SELECT r.reading_date, r.abnormal_reason, h.household_name, h.village_name
FROM readings r
JOIN households h ON r.water_meter_id = h.water_meter_id
WHERE r.is_abnormal = true
ORDER BY r.reading_date DESC
LIMIT :limit
```
