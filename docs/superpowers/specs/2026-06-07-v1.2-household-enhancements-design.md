# v1.2 村民管理增强 设计文档

> 日期：2026-06-07 | 状态：已批准

## 概述

在 v1.1 基础上增强村民管理模块：支持 Excel 批量导入/导出、详情页整合缴费历史。

## 改动清单

### 1. 批量导入/导出

**导出 API**：`GET /api/households/export`

- 参数：`villageNames` (选填), `waterMeterId` (选填)
- 逻辑：按筛选条件查活跃村民 → 生成 Excel → 返回下载
- Excel 列：户主姓名、联系电话、所属村名、水表编号、材料费总额

**导入 API**：`POST /api/households/import`

- 参数：`file` (MultipartFile, Excel 格式)
- 逻辑：解析 Excel → 全量唯一性校验（DB 已存在 + Excel 内部重复）→ 任何冲突即整批中止，返回冲突列表 → 全部通过后 `@Transactional` 写入，同时为每户创建材料费账单
- 返回：`{total: N, message: "导入成功"}` 或 `{conflicts: [...], message: "存在冲突"}`

### 2. 缴费历史集成

**前端**：`Households.vue` — 选中村民后，编辑表单下方展示缴费历史表格（从 `GET /api/payments/history` 获取数据）

**后端**：无改动，已有接口 `GET /api/payments/history?waterMeterId=xxx`

## 涉及文件

| 层 | 文件 | 改动类型 |
|----|------|---------|
| 后端 Controller | `HouseholdController.java` | 新增 `export`、`import` 两个接口 |
| 后端 Service | `HouseholdService.java` + `HouseholdServiceImpl.java` | 新增两个方法 |
| 前端 | `Households.vue` | 加导入/导出按钮 + 缴费历史表格 |

## 不涉及

- 无数据库表结构变更
- 无新增实体/Repository/DTO
- 无打包脚本修改
